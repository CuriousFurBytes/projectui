# Contributing

## Coding standards
- Use TypeScript strict typing; avoid `any`.
- Add or update tests for every behavior change.
- Follow red/green TDD: add failing test first, then implementation.
- Run `npm run typecheck`, `npm test`, and `npm run lint` before commit.

## Commit and PR
- Keep commits focused and descriptive.
- Include screenshots for visible UI changes.
- Document known limitations and migration impacts in PR description.
